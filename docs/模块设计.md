# 模块设计

## 设计目标

第一版遵循高内聚、低耦合原则：

- 领域模型只描述业务对象，不关心数据来自 JSON、CSV、数据库还是 API。
- 基础设施层负责数据读取和路径管理，不写分析逻辑。
- 服务层负责时间线、涨跌分析和报告生成，不直接依赖文件格式。
- CLI 只做参数解析和结果展示，不承载核心业务判断。

## 分层说明

### domain

路径：`src/finance_history/domain`

职责：

- 定义 `MoneyMaterial`、`Asset`、`MarketEvent`、`PricePoint`、`TrendWindow` 等核心对象。
- 定义 `PriceDirection` 等领域枚举。

### infrastructure

路径：`src/finance_history/infrastructure`

职责：

- 管理项目默认数据路径。
- 通过 `SeedRepository` 加载离线种子数据。
- 后续可新增数据库仓储、API 仓储或缓存仓储。

### services

路径：`src/finance_history/services`

职责：

- `TimelineService`：将货币材料和金融事件合并为时间线。
- `TrendAnalyzer`：分析资产起止价格变化、方向和关联事件。
- `MarkdownReportBuilder`：生成 Markdown 研究报告。

### cli

路径：`src/finance_history/cli.py`

职责：

- 提供 `summary`、`timeline`、`analyze`、`report` 命令。
- 保持薄入口设计，便于后续增加 Web、Notebook 或 API 层。

## 后续扩展接口

1. 数据源扩展：新增 `FredRepository`、`WorldBankRepository` 或 `ExchangeRepository`。
2. 分析扩展：新增波动率、最大回撤、相关性和事件窗口分析。
3. 可视化扩展：在服务层新增图表数据 DTO，再由前端或 Notebook 渲染。
4. 引用扩展：将 `source` 拆成独立引用表，支持 DOI、URL、访问日期和可信等级。
